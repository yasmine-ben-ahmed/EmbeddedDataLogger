145/******************************************************************************
146 * Supported ports
147 *
148 * TRC_HARDWARE_PORT_HWIndependent
149 * A hardware independent fallback option for event timestamping. Provides low
150 * resolution timestamps based on the OS tick.
151 * This may be used on the Win32 port, but may also be used on embedded hardware
152 * platforms. All time durations will be truncated to the OS tick frequency,
153 * typically 1 KHz. This means that a task or ISR that executes in less than
154 * 1 ms get an execution time of zero.
155 *
156 * TRC_HARDWARE_PORT_APPLICATION_DEFINED
157 * Allows for defining the port macros in other source code files.
158 *
159 * TRC_HARDWARE_PORT_Win32
160 * "Accurate" timestamping based on the Windows performance counter for Win32
161 * builds. Note that this gives the host machine time, not the kernel time.
162 *
163 * Hardware specific ports
164 * To get accurate timestamping, a hardware timer is necessary. Below are the
165 * available ports. Some of these are "unofficial", meaning that
166 * they have not yet been verified by Percepio but have been contributed by
167 * external developers. They should work, otherwise let us know by emailing
168 * support@percepio.com. Some work on any OS platform, while other are specific
169 * to a certain operating system.
170 *****************************************************************************/
171

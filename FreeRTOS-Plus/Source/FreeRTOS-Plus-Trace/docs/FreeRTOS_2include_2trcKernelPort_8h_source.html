1055 * The timestamps used in the recorder are "differential timestamps" (DTS), i.e.
1056 * the time since the last stored event. The DTS fields are either 1 or 2 bytes
1057 * in the other events, depending on the bytes available in the event struct.
1058 * If the time since the last event (the DTS) is larger than allowed for by
1059 * the DTS field of the current event, an XTS event is inserted immediately
1060 * before the original event. The XTS event contains up to 3 additional bytes
1061 * of the DTS value - the higher bytes of the true DTS value. The lower 1-2
1062 * bytes are stored in the normal DTS field.
1063 * There are two types of XTS events, XTS8 and XTS16. An XTS8 event is stored
1064 * when there is only room for 1 byte (8 bit) DTS data in the original event,
1065 * which means a limit of 0xFF (255UL). The XTS16 is used when the original event
1066 * has a 16 bit DTS field and thereby can handle values up to 0xFFFF (65535UL).
1067 *
1068 * Using a very high frequency time base can result in many XTS events.
1069 * Preferably, the time between two OS ticks should fit in 16 bits, i.e.,
1070 * at most 65535. If your time base has a higher frequency, you can define
1071 * the TRACE
1072 ******************************************************************************/
